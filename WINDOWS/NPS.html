/!\ ATTENTION/!\ Lors du transfert de config NPS les modèles ne suivent pas, il faut les exporter à part dans un fichier puis les importer sur le nouveau serveur après la migration de la configuration NPS

 

INSTALLER LE ROLE : "Service de stratégie et d'accès réseaux" ou en PS : Install-WindowsFeature NPAS -IncludeManagementTools

 

 

Pour exporter puis importer la configuration NPS en commande MAIS c'est aussi simple en graphique :

  1. Se mettre sur le NPS/NAP Source puis dans un PS taper :
    1. New-Item -Path '\\vdcsnps01\c$\NPSWConfigSettings' -ItemType Directory
    2. Export-NpsConfiguration –Path \\vdcsnps01\c$\NPSWConfigSettings\nps01.xml
  2. Se mettre sur le NPS Destination puis dans un ps taper :
    1. Import-NpsConfiguration –Path 'c:\NPSWConfigSettings\nps01.xml'

 

Pour inscrire un serveur NPS dans son domaine par défaut

  1. Sur le serveur NPS, dans Gestionnaire de serveur, cliquez sur Outils, puis sur serveur NPS ( Network Policy Server). La console du serveur de stratégie réseau s’ouvre.
  2. Cliquez avec le bouton droit sur NPS (local), puis cliquez sur inscrire le serveur dans Active Directory. La boîte de dialogue Serveur NPS (Network Policy Server) s’ouvre.
  3. Dans serveur de stratégie réseau, cliquez sur OK, puis de nouveau sur OK .

 

Pour annuler l’inscription d’un serveur NPS

  1. Sur le contrôleur de domaine, dans Gestionnaire de serveur, cliquez sur Outils, puis sur Active Directory les utilisateurs et les ordinateurs. La console Utilisateurs et ordinateurs Active Directory s'ouvre.
  2. Cliquez sur utilisateurs, puis double-cliquez sur serveurs RAS et IAS.
  3. Cliquez sur l’onglet membres , puis sélectionnez le serveur NPS dont vous souhaitez annuler l’inscription.
  4. Cliquez sur supprimer, sur Oui, puis sur OK.

 

Pour créer le certificat Serveur NPS par l'ADCS et l'enregistrer ds la config NPS : à terminer

  1. Lancer mmc sur le serveur nps
  2. Ajouter certificats sur le compte ordi local
  3. Dans personnel\certificats faire un clique droit et sélectionner nouvelle demande de certificats
  4. Suivant
  5. Puis cocher la case Serveur NPS
  6. Suivant
  7. Terminer
  8. Ouvrir la console NPS
  9. Aller dans stratégie\strategie reseau
  10. Clique droit propriétes sur Absyscyborg Wi-fi pareil sur l'autre
  11. Dans l'onglet contraintes sélectionner Microsoft PEAP puis modifier
  12. Valider ou sélectionner le bon certificat, regarder par la date car il a le même nom que le certif du serveur
  13. Tester le fonctionnement du Radius avec une vieille borne Wifi
  14. Mettre en prod sur les Meraki

POUR ACTIVER LES JOURNAUX D'AUDIT :
