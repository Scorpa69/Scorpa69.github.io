Problème de temps windows dans AD => PDC avec source de temps externe et le reste se synchro sur le PDC (Ne pas désactiver le service d'intégration hyper-v pour la date et l'heure) :

Sur tous les AD Virutels : reg add HKLM\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\VMICTimeProvider /v Enabled /t reg_dword /d 0

  • Sur le PDC : w32tm /config /manualpeerlist:“0.pool.ntp.org,0x1” /syncfromflags:MANUAL /reliable:yes
  • Sur le PDC : w32tm /config /update
  • Sur le PDC : w32tm /resync
  • Sur le PDC : w32tm /resync /rediscover
  • Sur les DC sauf le PDC : w32tm /config /syncfromflags:DOMHIER /update 
  • Sur les DC sauf le PDC : Restart-Service w32time ou net start "w32time"
  • Sur les DC sauf le PDC : w32tm /resync /force ou w32tm /resync /nowait

 

Voir la source et le status du NTP sur une machine :

  • w32tm /query /status
  • w32tm /query /source
  • w32tm /query /configuration
