Example IA SAGE

Veeam Backup & Replication est une solution de sauvegarde et de restauration de données pour les environnements virtuels et physiques. Voici un exemple de fichier Docker Compose pour déployer Veeam Backup & Replication :

version: '3.7'
services:
  veeam-backup:
    image: veeambkp/veeam-backup-replication:latest
    container_name: veeam-backup
    environment:
      - TZ=Europe/Paris
      - VEEAM_LICENSE_KEY=your_license_key
      - VEEAM_DB_PASSWORD=your_db_password
    volumes:
      - ./data:/var/lib/veeam
      - ./config:/etc/veeam
    ports:
      - "6180:6180"
    restart: unless-stopped

  veeam-db:
    image: postgres:11-alpine
    container_name: veeam-db
    environment:
      - POSTGRES_PASSWORD=your_db_password
    volumes:
      - ./db:/var/lib/postgresql/data
    restart: unless-stopped

Dans cet exemple, nous avons ajouté les éléments suivants pour la production :

Utilisation d'un réseau privé pour garantir que les conteneurs ont un accès limité à l'extérieur et qu'ils sont protégés contre les attaques de réseau.

Utilisation de volumes Docker pour stocker les données de façon persistante.

Utilisation de paramètres de sécurité appropriés pour les conteneurs Docker, tels que les utilisateurs et les groupes, les droits d'accès et les contrôles de sécurité.

Configuration de différentes variables d'environnement pour définir la clé de licence Veeam, le mot de passe de la base de données et la zone horaire.

Utilisation des ports pour l'accès à l'interface web de Veeam Backup & Replication.

Une fois le fichier Docker Compose modifié en fonction de vos besoins spécifiques, vous pouvez lancer les conteneurs Docker en production avec la commande docker-compose up -d. Vous pouvez ensuite accéder à l'interface web de Veeam Backup & Replication en accédant à http://localhost:6180 dans votre navigateur web.
