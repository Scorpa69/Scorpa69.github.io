Docker Compose file that uses an IPvlan L3 network mode for the MotionEyeOS container:

version: "3"

services:
  motioneye:
    image: ccrisan/motioneye:latest
    container_name: motioneye
    restart: always
    environment:
      - TZ=America/New_York
    ports:
      - 8765:8765
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /etc/motioneye:/etc/motioneye
      - /var/lib/motioneye:/var/lib/motioneye
      - /dev/video0:/dev/video0
    networks:
      motioneye_net:
        ipv4_address: 192.168.0.2

networks:
  motioneye_net:
    driver: ipvlan
    driver_opts:
      parent: eth0
      ipvlan_mode: l3
    ipam:
      driver: default
      config:
        - subnet: 192.168.0.0/24

In this Compose file, an IPvlan L3 network called motioneye_net is created with a subnet of 192.168.0.0/24. The MotionEyeOS container is configured to use this network and is assigned an IP address of 192.168.0.2.

To access the MotionEyeOS web interface, you can connect to http://192.168.0.2:8765 in your web browser. The MotionEyeOS container will be able to access the video device at /dev/video0 as long as it is mapped into the container as a volume.
