!!! TOUJOURS UTILISER L'ASSISTANT POUR LA CREATION DE PLAN DE MAINTENANCE !!!

VERIF EXISTANT

  1. Vérifier les historiques dans MSSMS ds AgentSQL server>Travaux et clique droit sur travaux puis séléctionner afficher l'historique : 

 

  1. Repérer tous les travaux/job en erreur
  2. Regarder l'erreur mentionné en cliquant sur pour déployer la sous tache puis cliquer sur la sous tache
  3. Afin d'afficher la totalité du message d'erreur, agrandir la fenêtre du bas ou est affiché les message d'erreur
  4. Analyse du message et si besoin aller corriger le job/travail comme indiqué à l'étape suivante
  5. Aller dans gestion>plan de maintenance et double clic sur le job/travail à vérifier (vérifier le chemin , verifier que le dossier existe par exemple)
  6. Pour les problème de taille de secteur disque, verifier la dernière backup dispo sur VVDAFIC01 et si ok supprimer le fichier présent sur le serveur (erreur qui apparait lors d'un P2V par exemple et plus généralement sur un changement de serveur)
  7. Pour les chemins inexistant, il faut les recéer avec les bon droits : utiliser les propriété de l'instance pour définir le dossier de sauvegarde par défaut avec les bon droit (clic droit sur l'instance puis propriétés et enfin sélectionner le menu paramètre de base de données)
  8. Ne pas oublier de sauvegarder le plan de maintenance si modif effectué
  9. Tester le plan de maintenance même en journée uniquement pour les sauvegardes, interdit de tester en journée pour les plan de maintenance d'optimisation 
  10. Vérifier la tache planifié move backup qui copie les sauvegarde sur VVDAFIC01 sur le serveur concerné

 

CREATION PLAN DE MAINTENANCE:

Toujours utiliser l'assistant de plan de maintenance en faisant un clic droit sur plan de maintenance puis assistant plan de maintenance

 

MAINTENANCE SUR SQL EXPRESS:

Pour SQL Express utilisation d'un script de "Ola Green" pour faire les plan de maintenance

https://ola.hallengren.com/

DROITS

Après avoir créer un plan de maintenance, il faut s'assurer que le propriétaire à les droits sur les bases de données sinon le plan sera en erreur :
 

 

Pour changer le propriétaire de la tâche/plan il suffit d'ouvrir les propriétés du travail dans AgentsSQL\Travaux\nomtravail clic droit sur le travail puis propriétes
