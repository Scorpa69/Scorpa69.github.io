OPTION DE LA COMMANDE FDISK :

   a  toggle a bootable flag

   b  edit bsd disklabel

   c  toggle the dos compatibility flag

   d  delete a partition

   l  list known partition types

   m  print this menu

   n  add a new partition

   o  create a new empty DOS partition table

   p  print the partition table

   q  quit without saving changes

   s  create a new empty Sun disklabel

   t  change a partition's system id

   u  change display/entry units

   v  verify the partition table

   w  write table to disk and exit

   x  extra functionality (experts only)

CREATION RAID 1 :

sudo mdadm --create /dev/md0 --level=raid1 --raid-devices=2 /dev/sdb /dev/sdc --spare-devices=1 /dev/sdd

sudo mdadm --query --detail /dev/md0 permet d'avoir l'UUID de la partition raid "md0"

sudo update-initramfs -u pour mettre à jour le fichier mdam.conf => Le système de démarrage de votre système a sa propre copie du fichier/etc/mdadm/mdadm.conf. Taper cette commande pour que les modifications sur ce fichier soient prises en compte au prochain démarrage

SIMULATION PANNE RAID  1 :

sudo mdadm --manage /dev/md0 --fail /dev/sdb   permet de simuler la panne de sdb

sudo mdadm --query --detail /dev/md pour voir l'état du raid et voir si le disque de spare "sdd" prend le relais en reconstruisant le RAID 1 dessus

sudo mdadm --manage /dev/md0 --remove /dev/sdb permet de retirer le disque sdb du RAID "md0"

sudo mdadm --manage /dev/md0 --add /dev/sdb permet de rajouter le nouveau disque qui a remplacer le disque "sdb" HS, une fois le RAID reconstruit sur le disque sdb, le disque sdd repasse en spare
