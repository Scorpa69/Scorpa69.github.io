Copier fichiers depuis local vers distant

scp <source> <user>@<host>:<destination>

Copier fichiers de distant vers local

scp <user>@<host>:<source> <destination>

Copier récursivement local => distant

scp -r <source> <user>@<host>:<destination>

Copier récursivement distant => local

scp -r <user>@<host>:<source> <destination>

Exemples :

scp C:\Users\A09\Nextcloud\Téléchargement\loaders.linux-x86_64.zip root@vdcstpm01:/root/loaders.linux-x86_64.zip

scp root@vdcszab01-dev:/root/zbx_db_partitiong.sql C:\Users\A09\Nextcloud\1-INFRA_AC\Projet\Zabbix\MySql\zbx_db_partitiong.sql

scp -r -p -4 'c:\users\a09\desktop\majlec\updates\root\' lively1953@dev-updates.compagnons-arcadie.com:/var/www/updates/root

scp lively1953@updates.compagnons-arcadie.com:/home/lively1953/majlecadmin 'C:\Users\A09\Downloads\majlecadmin'

Options SCP :

-l pour limiter la BP avec par exemple 1000 qui correspond à 1000Kb/s

-P pour préciser le port

-r est pour la copie récursive, qui comprendra tous les sous-répertoires.

-4 ou -6 peuvent être utilisés si vous souhaitez choisir la version de protocole utilisée, soit IPv4 ou IPv6.

-p préservera les temps de modification et les attributs initiaux du fichier.

-u supprimera le fichier source une fois le transfert terminé.

-c permettra la compression des données pendant l’opération de transfert
