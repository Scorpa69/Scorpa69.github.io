VOCABULAIRE AZURE

Scalability : Mise à l'échelle, Faire varier la puissance de calcul, la mémoire et le stockage d'une solution Azure

  • Vertical : variation du compute de la ram ou du stockage de la vm du service azure
  • Horizontal : variation du nombre de vm gérant le service azure

 

Elasticity : Capacité à faire du scaling automatique OU Mise à l'échelle automaique

 

Agility : Ajout/Suppression de ressources Azure manuellement OU Automatiquement par script

 

Fault Tolerance : Dans le Cloud c'est naturellement pris en charge dans tous les services Azure

 

Disaster Recovery : Restauration en cas de panne générale d'une région Azure entière, Est la capacité d'une solution Azure à pouvoir basculer d'une région à une autre en cas de panne régionale, on utilise la réplication du system/service/appli dans une autre régions avec un DNS qui redirige vers le système/service/appli disponible

 

High Availability : c'est le poucentage obtenu par => UPTIME/(UPTIME+DOWNTIME)

  • Beaucoup de service Azure fournissent par nature de la haute disponibilité, cela peut orienter vers tel ou tel solution selon ce critère

 

 

MODELE DE SERVICE CLOUD

  • ON PREMISE : On gère les trois
  • IAAS : Azure gère l'infra et on gère la Plateforme et le soft
  • PAAS : Azure gère l'infra et la plateforme et on gère le soft
  • SAAS : Azure gère tout et on loue le service (Office 365, Teams,….)
